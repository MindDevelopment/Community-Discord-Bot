# Changelog v1.1.5

**Release date:** 2026-07-23

---

## 🚀 Added

### Features
- `/setup-logs` command — one-time server setup tool (Admin only)
  - Creates 7 logging channels with emoji names and topics
  - Private channels (modlogs, reports, botlogs) hidden from `@everyone`
  - Skips channels that already exist
  - Self-disables after use via `SETUP_ENABLED` config flag
- `messageDelete` event — logs deleted messages to 📋-modlogs
- `messageUpdate` event — logs edited messages to 📋-modlogs (before/after + jump link)
- `guildMemberRemove` event — logs member leaves to 👋-vaarwel
- `hidden` property support for commands (excluded from `/help`)
- Category auto-detection for commands via folder name in commandHandler

### Infrastructure
- `SETUP_ENABLED` environment variable to control setup command visibility
- Graceful error reply when a registered command is not loaded (disabled)

---

## 🔄 Changed

### General
- `config.channels` defaults updated to emoji names matching `/setup-logs`
- `/help` command now dynamically generates command list from loaded commands
- `deploy-commands.js` skips `setup/` directory when `SETUP_ENABLED=false`
- `commandHandler.js` skips `setup/` directory when disabled

### Files
- `src/config.js` — added `setupEnabled`, updated `channels` defaults
- `src/commands/info/help.js` — fully rewritten to be dynamic
- `src/scripts/deploy-commands.js` — conditional setup directory loading
- `src/handlers/commandHandler.js` — conditional setup loading + category property
- `src/events/interactionCreate.js` — graceful handling of disabled commands
- `.env.example` — added `SETUP_ENABLED` documentation

---

## 🛠 Fixed

### Bugs
- `/help` command had hardcoded command list that was out of sync with actual commands
- Running a previously-registered but disabled command would silently timeout (no response)
- Commands had no `category` property set, making dynamic grouping impossible

---

## 📖 Documentation

- `.env.example` updated with `SETUP_ENABLED` variable and description
- `Dev/project-upgrade.md` — comprehensive upgrade plan created

---

## 🗂 Database

### Added
- `settings` entry `setup_completed` (per guild) — set to `true` after `/setup-logs` runs

---

## 📌 Notes

- After running `/setup-logs`, set `SETUP_ENABLED=false` in `.env` and re-deploy to hide the command from the slash command list.
- To re-run setup, set `SETUP_ENABLED=true`, delete the `setup_completed` setting from the database, and re-deploy.
